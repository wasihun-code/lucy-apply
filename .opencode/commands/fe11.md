# /fe11 — Admin Portal Shell + Universities Management

## What you are building in this sprint

The platform admin experience: universities list, university detail/edit, and onboarding
a new university. The admin portal uses `StaffShell` with a platform-admin variant nav.

## Dependency check

FE-07 (StaffShell with role-aware nav). Admin nav items (Universities, Users, Stats,
Audit Log) must be wired in StaffShell for `platform_admin` role.

---

## Deliverables

### 1. Verify StaffShell Admin Nav

In `StaffShell`, when `role === 'platform_admin'`, the sidebar shows:
- Universities (`/admin/universities`)
- Users (`/admin/users`)
- Stats (`/admin/stats`)
- Audit Log (`/admin/audit-log`)

No university name header (platform admin is cross-tenant).
Instead: "Lucy Apply Admin" in the sidebar header.

---

### 2. Admin Index Redirect

`frontend/app/admin/page.tsx` → redirect to `/admin/universities`.

---

### 3. Universities List — `frontend/app/admin/universities/page.tsx`

Client component. Fetches `GET /api/v1/admin/universities/` (unscoped, all universities).

`<PageHeader title="Universities" action={<Button variant="primary" size="sm" href="/admin/universities/new">+ Onboard University</Button>} />`

**Filter bar:** Status dropdown: All | Active | Inactive

**Universities table:** `<Table>` with columns:

| Column | Content |
|---|---|
| Name | University name (link → `/admin/universities/{id}`) |
| Status | `<StatusBadge status={status}>` |
| Programs | Program count (from API annotations) |
| Applications | Application count |
| Created | `formatDate(created_at)` |
| Actions | Activate / Deactivate button |

Activate: `PATCH /api/v1/universities/{id}/status/` with `{ status: 'active' }`
Deactivate: same endpoint with `{ status: 'inactive' }` — requires confirmation modal.

Row click → `/admin/universities/{id}`

**Empty state:** `<EmptyState icon={<Building2>} heading="No universities" description="Onboard your first university partner to get started." action={{ label: '+ Onboard University', href: '/admin/universities/new' }} />`

---

### 4. University Detail — `frontend/app/admin/universities/[id]/page.tsx`

Client component. Fetches `GET /api/v1/universities/{id}/`.

`<PageHeader title={university.name} breadcrumb={[{ label: 'Universities', href: '/admin/universities' }, { label: university.name }]} action={<Button variant="secondary" size="sm">Edit</Button>} />`

Two sections:

**University Info card:**
- Name, Description, Accreditation Info (editable via inline edit or separate form)
- Status badge
- Created date
- Stats: program count, application count

**Quick Actions card:**
- Activate / Deactivate status toggle button (with confirmation)
- "View Programs" → filters `/admin/universities` to show this uni's programs
  (or links to `/portal/programs` — note: admin can view but not manage via portal)

Keep this page lightweight — full edit is via the Edit button route.

---

### 5. Onboard University Form — `frontend/app/admin/universities/new/page.tsx`

`<PageHeader title="Onboard University" breadcrumb={[{ label: 'Universities', href: '/admin/universities' }, { label: 'New' }]} />`

Form inside `<Card padding="lg">`:

```
Section: "University Details"
  Name *                  <Input>
  Description             <Textarea rows={3}>
  Accreditation Info      <Textarea rows={3} hint="Accreditation body, certification details">

Section: "Initial Admin Account"
  Admin Full Name *       <Input>
  Admin Email *           <Input type="email">
  (Admin will receive an invitation email to set their password)

Footer:
  <Button variant="primary">Create University & Send Invitation</Button>
  <Button variant="secondary" href="/admin/universities">Cancel</Button>
```

API: `POST /api/v1/universities/` with:
```json
{
  "name": "...",
  "description": "...",
  "accreditation_info": "...",
  "initial_admin_email": "...",
  "initial_admin_name": "..."
}
```

On success:
```
<Alert variant="success">
  University created. An invitation has been sent to {admin_email}.
  The university is currently inactive until the admin completes setup.
</Alert>
```
+ "View Universities" button. Do NOT auto-redirect — keep the success message visible.

On error: `<Alert variant="danger">{error}</Alert>` above form.
