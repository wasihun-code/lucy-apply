# /fe03 — University Detail + Program Detail

## What you are building in this sprint

The two most important public-facing pages for conversion: the university profile and
the program detail. An applicant who lands on either of these pages should have all
the information they need to decide to apply, and a clear path to do so.

## Dependency check

FE-01 (design system) and FE-02 (UniversityCard, ProgramCard components) must be complete.

---

## Deliverables

### 1. University Detail — `frontend/app/universities/[id]/page.tsx`

Server component. Fetches:
- `GET /api/v1/universities/{id}/` → university data
- `GET /api/v1/programs/?university={id}` → programs for this university

**Layout sections:**

**Hero band:**
- Background: `bg-primary-soft` with subtle pattern or just clean color
- University logo: if present, show `<Image>` (80×80, rounded-lg, white bg, shadow-sm);
  else initials avatar (64px, `bg-primary text-white`, rounded-xl)
- University name: `text-4xl font-display font-bold text-text-900`
- One-line description below name: `text-lg text-text-600`
- Status badge: only show if inactive (`<StatusBadge status="inactive">` with
  "Applications Paused" label)

**Info bar:**
- Below hero, above content
- Stats row: number of programs, number of open cycles (computed from programs data)
- `text-sm text-text-600` labels with `text-base font-semibold text-text-900` values

**About section:**
- Heading: "About"
- `accreditation_info` if present, else description
- `text-sm text-text-600 leading-relaxed`

**Programs section:**
- Heading: "Programs" with program count badge
- Filter tabs: All | Undergraduate | Postgraduate (client component)
- Program list: `<ProgramCard>` components (from FE-02) in a vertical list (not grid)
  on this page — more scannable with longer descriptions
- Empty state if no programs: `<EmptyState heading="No programs available" />`

**Sidebar (on md+ screens):**
- Sticky sidebar with:
  - "Start Your Application" heading
  - "Browse our programs below to apply" body text
  - `<Button variant="primary" className="w-full">Browse Programs</Button>` (scroll to programs)
  - Divider
  - Contact/link placeholder (structure only, no real data)

### 2. Program Detail — `frontend/app/universities/[id]/programs/[programId]/page.tsx`

Server component. Fetches:
- `GET /api/v1/programs/{programId}/` → program + required_documents + open cycles

**Breadcrumb:** Universities → {university name} → {program name}
Implement using `<PageHeader breadcrumb=[...]>` component.

**Layout: two-column on lg+ (main content left, apply sidebar right)**

**Main content (left column):**

Program header:
- Degree badge: `<StatusBadge>` with degree_level as label
- Program name: `text-3xl font-display font-bold`
- University name link: → university detail page

Description section:
- Heading: "About This Program"
- `program.description` (or placeholder if empty)
- `program.requirements` as a separate "Requirements" section if present

Required Documents section:
- Heading: "Required Documents"
- Checklist list: each document as a row with a file icon (lucide `FileText`) and label
- `text-sm text-text-600` per item

**Apply Sidebar (right column, sticky on desktop):**
Card with:
- Application fee: `text-2xl font-bold text-text-900` + `text-sm text-text-400 "USD"` label
- Section: "Open Admission Cycles"
  - For each open cycle: cycle name, deadline date, "Apply Now" button
    → `/dashboard/apply/{programId}?cycleId={cycle.id}` (or prompt to login if not auth)
  - If no open cycles: `<Alert variant="info">No open cycles at this time.</Alert>`
- "Save for later" ghost button (UI only, no backend — future feature placeholder)

**What changes from the current page:**
- Current page has a bare `<h1>`, a fee as plain text, a bullet list of documents,
  and a single "Apply Now" row per cycle. All of this is functionally correct but visually bare.
- This sprint adds the two-column layout, breadcrumb, section headings, the document
  checklist styling, and the styled sidebar.
- Do not change the Apply Now routing logic — just style it.
