### 4. Step 2 — Documents

**Keep:** upload URL generation, document registration, re-upload logic.
**Redesign:** document upload UI.

Section heading: "Required Documents"
Subtext: "Upload the required files for this program. PDF, JPG, PNG, DOC, DOCX accepted. Max 10MB per file."

Each document as a `<Card padding="md">` row:

```
[FileIcon] Document Type Label          [StatusBadge]
           Status description           [Upload Button]
```

States per document:
- **Not uploaded:** `<Button variant="secondary" size="sm">Upload File</Button>`
  Opens a file input. Shows file name + size after selection before upload.
- **Uploading:** `<Spinner size="sm" />` + "Uploading..."
- **Pending review:** `<StatusBadge status="pending" />` + `<Button variant="ghost" size="sm">Re-upload</Button>`
- **Verified:** `<StatusBadge status="active" />` (green) — no upload button
- **Flagged:** `<StatusBadge status="rejected" />` + flagged reason in `text-sm text-danger` + `<Button variant="secondary" size="sm">Re-upload</Button>`

File size enforcement: check `file.size > 10 * 1024 * 1024` before calling upload-url.
Show `<Alert variant="danger">File must be under 10MB.</Alert>` inline if too large.

Navigation:
- `<Button variant="secondary">← Back to Form</Button>` left
- `<Button variant="primary">Next: Review & Pay →</Button>` right
- Disable "Next" if required documents are not all uploaded (any status — just must exist)

### 5. Step 3 — Review & Submit

**Keep:** payment intent logic, submit API call.
**Fix BUG-FE-001:** Raw JSON error must never reach the DOM.
**Redesign:** presentation + error handling.

Section heading: "Review & Submit"

Summary cards (3 stacked `<Card padding="md">` components):

**Program card:**
- Label: "Program"
- Program name + University name

**Documents card:**
- Label: "Documents"
- List: each document + `<StatusBadge status={doc.status} />`
- If any flagged: `<Alert variant="warning">Some documents are flagged. You can still submit but they may need attention.</Alert>`

**Fee card:**
- Label: "Application Fee"
- Amount: `text-2xl font-bold` + currency label
- Note: "Non-refundable" in `text-xs text-text-400`

**Error handling (BUG-FE-001 fix):**
```tsx
const [submitError, setSubmitError] = useState<string | null>(null)

try {
  await api('/applications/{id}/payment-intent/', { method: 'POST' })
} catch (e) {
  if (e instanceof ApiError) {
    // Map known error codes to human-readable messages
    const messages: Record<string, string> = {
      CYCLE_CLOSED:     'The application deadline has passed for this program.',
      MISSING_DOCS:     'Please upload all required documents before submitting.',
      PAYMENT_REQUIRED: 'Payment is required to submit your application.',
    }
    setSubmitError(messages[e.body?.code] ?? e.message)
  } else {
    setSubmitError('Something went wrong. Please try again.')
  }
}

// In JSX — above the buttons:
{submitError && <Alert variant="danger">{submitError}</Alert>}
```

**NEVER:** `setSubmitError(JSON.stringify(e))` or `setSubmitError(String(e))`

Submit button: `<Button variant="primary" size="lg" loading={isSubmitting}>Pay {formatCurrency(fee)} & Submit</Button>`

Navigation: `<Button variant="secondary">← Back to Documents</Button>` left
