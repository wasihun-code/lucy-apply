### 3. Applicant Profile — `frontend/app/dashboard/profile/page.tsx`

Client component. Fetches `GET /api/v1/applicants/me/`.

**Page header:** `<PageHeader title="My Profile" />`

Two cards:

**Personal Information card:**

Section heading: "Personal Information"

Display mode (default): definition list of current values with an "Edit" button top-right.
Edit mode (on "Edit" click): form fields pre-populated, "Save Changes" + "Cancel" buttons.

Fields:
- Full Name
- Email (read-only — changing email has security implications, show note)
- Country of Residence
- Date of Birth
- Nationality

Edit API: `PATCH /api/v1/applicants/me/` with changed fields.
On success: return to display mode with updated values + `<Alert variant="success">Profile updated.</Alert>`
On error: `<Alert variant="danger">{error}</Alert>`

Email is shown but not editable inline. Below the email field (in edit mode):
`<p className="text-xs text-text-400">To change your email, contact support.</p>`

**Account Security card:**

Section heading: "Account Security"

```
Password
Last changed: [date or "Unknown"]
[Change Password] button → opens modal

Two-Factor Authentication
Status: Enabled / Not enabled
[Manage 2FA] button → /mfa/setup
```

**Change Password modal:**

```
<Modal title="Change Password">
  <FormField label="Current Password" required>
    <Input type="password" />
  </FormField>
  <FormField label="New Password" required hint="Minimum 8 characters">
    <Input type="password" />
  </FormField>
  <FormField label="Confirm New Password" required>
    <Input type="password" />
  </FormField>
  {error && <Alert variant="danger">{error}</Alert>}
  [Cancel] [Update Password]
</Modal>
```

Client-side: new password and confirm must match before submit.
API: `POST /api/v1/auth/reset-password/` with `{ password, current_password }` — or
whichever endpoint exists for authenticated password change. Check the backend.
On success: close modal + `<Alert variant="success">Password updated.</Alert>`
