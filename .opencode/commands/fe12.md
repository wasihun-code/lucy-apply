# /fe12 — Admin: Users + Audit Log

## What you are building in this sprint

Two admin pages that expose backend capabilities with no current UI:
the cross-model user management page and the filterable audit log.

## Dependency check

FE-11 (admin shell wired). Backend: `/admin/users/`, `/admin/audit-log/`,
`/universities/{id}/audit-log/` all working from sprint 9.

---

## Deliverables

### 1. Users Management — `frontend/app/admin/users/page.tsx`

Client component. Fetches `GET /api/v1/admin/users/` with search params.

`<PageHeader title="Users" />`

**Search + filter bar:**
- Search input: `<Input placeholder="Search by name or email..." />` — debounced 300ms, passes `?search=` to API
- Role filter: dropdown — All | Applicants | University Staff | Platform Admins
- Status filter: dropdown — All | Active | Deactivated

**Users table:** `<Table>` with columns:

| Column | Content |
|---|---|
| Name | Full name + avatar initial |
| Email | Email address |
| Role | `<StatusBadge>` — "Applicant", "Staff", "Admin" |
| University | University name for staff, "—" for others |
| Status | `<StatusBadge status={account_status}>` |
| Actions | Deactivate button |

Avatar initial: circle div, 32px, `bg-primary-soft text-primary text-xs font-semibold`
containing first letter of full_name.

Deactivate: `<Button variant="ghost" size="sm"><UserX size={16} className="text-danger" /></Button>`
Only shown if status is `active`.
Confirmation modal: "Deactivate {name}? They will immediately lose access to the platform."
API: `PATCH /api/v1/admin/users/{id}/status/` with `{ status: 'deactivated' }`
On success: update row status badge optimistically.

Cannot deactivate your own account (compare with `getMe().id`).

**Empty state:** `<EmptyState icon={<Users>} heading="No users found" description="Try adjusting your search or filters." />`

**Pagination:** `<Pagination>` if > 20 results.

### 2. Platform Audit Log — `frontend/app/admin/audit-log/page.tsx`

Client component. Fetches `GET /api/v1/admin/audit-log/` (unscoped, all events).

`<PageHeader title="Audit Log" />`

**Filter bar:**
- University select: All universities + list from `/admin/universities/`
- Action type select: All | Application Status Change | Document Flagged | Document Verified | Payment | University Onboarded | Staff Account | Decision Issued | Decision Reversed
- Date range: Start date + End date (`<Input type="date">` × 2)
- `<Button variant="secondary" size="sm">Clear Filters</Button>`

**Audit log table:** `<Table>` with columns:

| Column | Content |
|---|---|
| Time | `formatDate(created_at)` + time |
| Actor | actor_type badge + actor_id (truncated, monospace) |
| Action | Human-readable action string |
| Entity | entity_type + entity_id (truncated, monospace) |
| University | University name or "Platform" |

Action → human label mapping:
```
application_status_change → "Application Status Changed"
document_flagged          → "Document Flagged"
document_verified         → "Document Verified"
payment_completed         → "Payment Completed"
university_onboarded      → "University Onboarded"
staff_account_created     → "Staff Invited"
staff_account_deactivated → "Staff Deactivated"
decision_issued           → "Decision Issued"
decision_reversed         → "Decision Reversed"
```

Row expandable: clicking a row expands to show `before_state` and `after_state` JSON
in a `<pre className="font-mono text-xs bg-background p-3 rounded overflow-auto">`.
(Not a modal — inline expand within the table row.)

**Empty state:** `<EmptyState icon={<ScrollText>} heading="No audit events" description="Audit events will appear here as actions are performed." />`

**Pagination:** Always paginated (audit log can be large).

---

### 3. University Audit Log — `frontend/app/portal/audit-log/page.tsx`

Same design as platform audit log but:
- Fetches `GET /api/v1/universities/{id}/audit-log/` (scoped to the staff's university)
- No university filter (it's always the current university)
- Accessible to `permission_level === 'admin'` (not officer)
- Add to StaffShell admin nav: "Audit Log" → `/portal/audit-log`
