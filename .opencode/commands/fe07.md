### 3. Application Review Queue — `frontend/app/portal/applications/page.tsx`

Client component. Fetches:
`GET /api/v1/universities/{university_id}/applications/?status=&program=`

University ID read from `getMe()` → `university_id`.

**Page header:** `<PageHeader title="Applications" />`

**Filter bar** (above table, inline row):
- Status select: All | Draft | Submitted | Under Review | Admitted | Rejected | Waitlisted | Accepted | Declined
- Program select: All Programs + list from `GET /api/v1/universities/{id}/programs/`
- Both update URL query params (shareable/bookmarkable)
- `<Button variant="secondary" size="sm">Clear Filters</Button>` — resets both

**Results table:** `<Table>` component with columns:

| Column | Content |
|---|---|
| Applicant | Name (bold) + email (text-400, smaller) |
| Program | Program name |
| Submitted | Date or "—" if draft |
| Documents | "X/Y verified" progress |
| Status | `<StatusBadge>` |
| Action | "Review →" link button |

- Default sort: submitted date descending
- Clicking column headers: sort asc/desc (client-side for current page data)
- Row hover: `bg-primary-soft/20`
- Clicking a row navigates to `/portal/applications/{id}`

**Empty state:**
`<EmptyState icon={<Inbox>} heading="No applications" description="Applications submitted to your university will appear here." />`

**Loading:** Table skeleton — header row + 5 `<SkeletonRow />` components.

**Pagination:** if > 20 results, show `<Pagination>` component.
Pass `?page=` query param to the API.
