# /fe02 — Landing Page + University Listing

## What you are building in this sprint

The public-facing discovery experience. A landing page that actually sells the product,
and a university listing that lets applicants find what they're looking for.

## Dependency check

FE-01 must be complete: design tokens, all three layout shells, and base components
must exist before touching any page content.

---

## Deliverables

### 1. Landing Page — `frontend/app/page.tsx`

Replace the current minimal hero with a structured landing page.
All content is server-rendered (SSC). Fetch universities from `GET /api/v1/universities/`.

**Section 1: Hero**
- Headline: "Apply to Ethiopian Universities — All in One Place"
- Subheadline: "Browse programs, submit your application, and track your admission
  status from anywhere in the world."
- Primary CTA: `<Button variant="primary" size="lg">Browse Programs</Button>` → `/universities`
- Secondary CTA: `<Button variant="secondary" size="lg">How It Works</Button>` → scrolls to #how-it-works
- Background: `bg-background` (not white, not image — keep it clean)
- Layout: full-width section, `py-20`, content max-width centered

**Section 2: Featured Universities**
- Heading: "Partner Universities"
- Grid: 2-col on sm, 3-col on lg
- Show first 6 active universities from API
- Each university: `<UniversityCard>` component (see below)
- "View all universities →" link below grid

**Section 3: How It Works** (id="how-it-works")
- 3-step flow: Browse Programs → Apply Online → Track Your Status
- Each step: icon (lucide) + number + title + one-sentence description
- Layout: 3-column grid on md+, stacked on mobile

**Section 4: CTA Banner**
- "Ready to start your application?"
- Single button: "Create Free Account" → `/register`
- Background: `bg-primary-soft` band

### 2. UniversityCard Component — `frontend/components/shared/UniversityCard.tsx`

Used on landing page and university listing. NOT a link — wraps in Next.js `<Link>`.

```tsx
type UniversityCardProps = {
  university: {
    id: string
    name: string
    description: string
    status: string
    logo?: string | null
  }
}
```

Layout:
- Logo slot: if `logo` exists, show `<Image>` (40×40, rounded); else show initials avatar
  (first 2 letters of university name, `bg-primary-soft text-primary`, rounded-full)
- University name: `text-base font-display font-semibold text-text-900`
- Description: `text-sm text-text-600 line-clamp-2`
- Footer: "View Programs →" in `text-sm text-primary`
- Card: `<Card interactive>` — full card is clickable via Link
- Hover: shadow-md + border-primary/30

### 3. University Listing — `frontend/app/universities/page.tsx`

Server component. Fetches `GET /api/v1/universities/`.

**Page header:** `<PageHeader title="Universities" description="Browse our partner universities and explore their programs." />`

**Search + filter bar:**
- Search input (client-side filter by name, `<Input>` component)
- No server-side search needed at this scale — filter the fetched list client-side
- The search bar requires a thin Client Component wrapper — use the pattern:
  ```tsx
  // UniversityListClient.tsx (client)
  // receives universities[] as prop from server component
  // handles search state locally
  ```

**Results:** Grid of `<UniversityCard>` — same component as landing page.
2-col sm, 3-col lg.

**Empty state:** `<EmptyState icon={<Building2>} heading="No universities found" description="Try a different search." />`

**Loading:** Use `loading.tsx` next to `page.tsx` with a grid of `<SkeletonCard />` components.

### 4. Update Public Programs Page — `frontend/app/programs/page.tsx` (NEW ROUTE)

Create this new route. Backend: `GET /api/v1/programs/?degree_level=&university=`

This is the new cross-university program discovery page that currently does not exist.

**Page header:** `<PageHeader title="Programs" description="Find the right program across all our partner universities." />`

**Filter bar (client component):**
- Degree level: dropdown select (`all | undergraduate | postgraduate`)
- University: dropdown select (all universities, populated from API)
- Filters update URL query params for shareability

**Results:** List of `<ProgramCard>` components.

#### ProgramCard Component — `frontend/components/shared/ProgramCard.tsx`

```tsx
type ProgramCardProps = {
  program: {
    id: string
    name: string
    degree_level: string
    description: string
    fee_amount: string
    fee_currency: string
    university: { id: string; name: string }
    open_cycles?: { close_date: string }[]
  }
}
```

Layout:
- University name: `text-xs text-text-400 uppercase tracking-wide`
- Program name: `text-base font-display font-semibold text-text-900`
- Degree badge: `<StatusBadge status="published">` with degree_level label override
- Fee: `text-sm text-text-600` — use `formatCurrency(fee_amount, fee_currency)`
- Deadline: if open cycle exists, "Deadline: {close_date formatted}" in `text-sm text-warning`
  else "Applications closed" in `text-sm text-text-400`
- CTA: "View Program →" link, right-aligned

**Empty state:** `<EmptyState heading="No programs found" description="Try adjusting your filters." />`

**Update nav:** Add "Programs" link to `PublicShell` navbar pointing to `/programs`.
