# /fe13 — Admin Stats Dashboard

## What you are building in this sprint

The platform admin stats dashboard — the first page an admin sees that gives them a
real-time health view of the platform. Backend: `GET /api/v1/admin/stats/`.

## Dependency check

FE-11 (admin shell). The stats endpoint must be working (confirmed in sprint 9 backend).

---

## Deliverables

### 1. Stats Dashboard — `frontend/app/admin/stats/page.tsx`

Client component. Fetches `GET /api/v1/admin/stats/`.

`<PageHeader title="Platform Overview" />`

**Summary stat cards — row of 4:**

| Stat | Label | Icon |
|---|---|---|
| Total Applicants | "Registered Applicants" | Users |
| Total Universities | "Partner Universities" | Building2 |
| Total Applications | "Total Applications" | FileText |
| Active Cycles | "Open Admission Cycles" | Calendar |

Each card: `<Card>` with:
- Icon in `bg-primary-soft text-primary rounded-lg p-2` — top-left
- Large number: `text-4xl font-display font-bold text-text-900`
- Label: `text-sm text-text-400`
- Subtle trend indicator if data available (e.g. "+3 this month" in `text-xs text-success`)

**Applications by Status — breakdown cards row:**

| Status | Count |
|---|---|
| Draft | count |
| Submitted | count |
| Under Review | count |
| Admitted | count |
| Rejected | count |
| Accepted | count |

Display as a horizontal bar or a grid of small stat chips.
Each chip: `<StatusBadge status={status}>` + count number.
Use the same semantic colors as badges — so the visual is consistent.

Simple implementation: 6 small `<Card padding="sm">` in a 3×2 grid:
```
[StatusBadge] [Count large]
[Label small]
```

**Universities Section:**

Heading: "Universities" with "View all →" link to `/admin/universities`.

Small table showing top 5 universities by application count:
- University name
- Status badge
- Application count
- Program count

No pagination needed here — just top 5 as a preview.

**Recent Activity (optional, if API supports it):**

If `GET /admin/audit-log/` returns recent events without heavy filtering,
show the last 5 audit events as a simple list:
- Event action label
- University name
- Time (relative: "2 hours ago")
- "View all →" link to `/admin/audit-log`

If the audit log endpoint is too heavy for this, skip this section.
